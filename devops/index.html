WHAT ?

- different concerns
- different cultures
- dev = product ( improvements / features )
- ops = service ( backups / supervision )
- same global objective :
-- system working from outside -- client satisfaction - those are the "original" facts - DevOps is not limiting to that, wants to be open
A problem occuring between devs and ops might find it's solution elsewhere
-Think about the wall
=> a business throwing specs and sold stuff to devs
=> a QA throwing a "bug" back to devs
etc ...

But not only !
